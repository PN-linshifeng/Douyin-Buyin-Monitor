# 监听接口产品列表接口，这个逻辑代码写在product_list.js中
url：/pc/selection/common/material_list

body参数：{
    "scene": "PCSquareFeed",
    "size": 0,
    "search_text": "",
    "cursor": 0,
    "extra": {
        "new_session_strategy": "1",
        "search_id": "",
        "session_id": "",
        "use_kol_product": "1"
    },
    "filters": {}
}
返回的数据如下：

{
  data:{
    summary_promotions:[
      {
        "product_id": "3790911781329829972",
        "promotion_id": "3790936756430438874",
        "base_model": {
            "product_info": {
                "name": " 专属【买一送一】全脂 新西兰奶源高钙高蛋白牛奶粉 0蔗糖1kg/罐",
            },
        }
      }
    ]
  }
}

把请求返回的数据data.summary_promotions[]追加保存起来，如果body的cursor等于0时，清空保存的数据,保存最新的数据；

## UI和事件点击需求
这个时候页面会更新最新的数据，获取class为"index_module__wrapper___dadac"的元素，在里面新增一个按钮“获取选品数据”，老数据可能已经新增了按钮。点击后，获取子元素class为"element-timing"的元素的文本，在根据文本内容从保存的数据中找出对应的promotionId，然后请求product_info接口，元素的文本=base_model.product_info.name,promotionId=promotion_id
